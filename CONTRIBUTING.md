# Contributing

Pull requests are welcome for improvements to the Markdown skills, documentation, examples, research references, release validation, and safety guidance.

Before making a substantial change to the CLOUD framework itself, open an issue so the proposal can be discussed with Dr Lindsay Demers and the maintainers.

When contributing:

- Explain whether the change affects CLOUD, another skill, supporting research, documentation, or release tooling.
- Cite new research in `SOURCES.md` and the relevant `REFERENCE.md`.
- Do not copy wording, assessment items, diagrams, or scoring material unless its licence or written permission allows reuse.
- Identify whether contributed material is original, public domain, or third-party licensed.
- Preserve creator, copyright, licence, and modification notices.
- Regenerate and validate release metadata when skill content changes.
- Add or update tests, examples, and safety guidance where relevant.

By submitting a contribution, you confirm that you have the right to contribute it. Unless a separate written agreement applies, content contributions are submitted under CC BY 4.0 and code contributions under Apache-2.0, matching the outbound licence for the relevant part of the repository.
